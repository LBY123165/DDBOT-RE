# DDBOT-RE 重构方案

## 一、重构目标

1. **模块化架构**：将每个平台作为独立模块运行，实现模块的注册、启动、停止和热更新
2. **热更新系统**：检测模块更新并通知管理员，支持模块的动态更新
3. **WebUI 整合**：将 WebUI 整合到程序中，启动时同时启动 WebUI
4. **数据库更换**：放弃 buntdb，使用更加方便的数据库
5. **代码优化**：优化代码结构，提高可维护性和可读性

## 二、架构设计

### 2. 核心组件

#### 2.1 模块系统

- **Module 接口**：定义模块的基本方法，包括启动、停止、重载等
- **ModuleManager**：负责模块的注册、启动、停止和热更新
- **平台模块**：实现各个平台的具体功能，如B站、A站、YouTube等

#### 2.2 热更新系统

- **UpdateManager**：负责检查模块更新、通知管理员和执行更新
- **更新流程**：检查更新 → 通知管理员 → 下载模块 → 解压模块 → 替换模块 → 重启模块

#### 2.3 WebUI 整合

- **WebUIService**：提供 WebUI 静态文件服务
- **自动启动**：在应用启动时自动启动 WebUI 服务

#### 2.4 应用核心

- **App**：应用实例，整合所有组件
- **启动流程**：初始化Bot → 注册服务 → 注册模块 → 启动服务 → 启动模块 → 启动命令管理器 → 启动关注管理器 → 启动热更新管理器

## 三、实现步骤

### 3. 实现热更新系统

1. **实现更新管理器**：实现 UpdateManager，支持检查更新和执行更新

### 5. 数据迁移

1. **实现数据迁移工具**：实现从原有 buntdb 数据迁移到新 SQLite 数据库的功能

### 6. 优化代码结构

1. **优化导入结构**：整理导入包，去除不必要的导入
2. **优化代码风格**：统一代码风格，提高可读性
3. **添加注释**：为关键代码添加注释，提高可维护性
4. **保持代码仓库不变**：确保 go.mod 文件中的模块路径保持为 github.com/cnxysoft/DDBOT-WSa

## 四、技术实现

### 1. 模块管理

- 使用 `map` 存储模块，支持动态注册和管理
- 使用 `sync.RWMutex` 保证并发安全
- 支持模块的启动、停止和重载
- 参考 DDBOT-WSa 目录中的原有代码，实现各平台模块的具体功能

### 2. 热更新

- 使用 `time.Ticker` 定期检查更新
- 使用 `http.Get` 下载模块
- 使用文件操作解压模块
- 支持动态替换模块实例

### 3. WebUI

- 使用 `http.FileServer` 提供静态文件服务
- 使用 `http.ListenAndServe` 启动 HTTP 服务器
- 支持多个可能的 WebUI 路径
- 参考 DDBOT-WSa-Desktop 目录中的原有代码，确保 WebUI 功能完整

### 4. 数据库

- 使用 `database/sql` 包和 `github.com/mattn/go-sqlite3` 驱动实现 SQLite 数据库
- 实现数据库初始化、连接和操作
- 实现数据迁移工具，从原有 buntdb 数据迁移到新 SQLite 数据库

### 5. 并发处理

- 使用 `goroutine` 处理模块的启动和运行
- 使用 `sync.WaitGroup` 等待所有模块和服务停止
